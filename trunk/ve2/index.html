ve2 - the voiD editor is developed and maintained by the Linked Data Research Centre, DERI, NUI Galway and uses the following services:

  • The DBpedia look-up service
  • The Talis voiD store
  • The RKB voiD store
  • The namespace lookup prefix.cc for RDF developers

The code is available via Google code. In case you have a feature request or want to file a bug, please visit the issue tracker.

Kudos to Richard and Lin for early feedback and lots of good suggestions how to improve the usability ;)

Interlink to other datasets

Provide the target dataset by specifying the homepage of the dataset you want to link to. The homepage of the dataset is a dedicated document that talks about the dataset. Use the 'peek' button to preview known datasets from voiD stores (currently Talis and RKB).
For the interlinking type, you need to provide an RDF property such as http://www.w3.org/2002/07/owl#sameAs (note: you can type prefixes such as owl or foaf and then use the 'lookup' button to replace it with the fully qualified namespace URI); it's also o.k. to use the 'short form', such as owl:sameAs, which will be replaced automatically. For more information see the voiD guide, section 'Describing Dataset Interlinking'.

Tell about provenance and licensing

State who publishes the data and what the usage conditions are. For the publisher, provide a URI identifying the person or organisation (Web ID) that actually provides the dataset in RDF. The source is the URI of the original data source (such as an XML document, a database, a Web service, etc.). The license should be specified via an URI representing the license under which the dataset can be used. If you don't want to provide any of this information, just leave it empty - it's optional, but it is advised to provide it. For more information see the voiD guide, section 'General Dataset Metadata' and 'Announcing the license of a dataset'.
